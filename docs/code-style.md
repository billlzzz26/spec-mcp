# Code Style Guidelines

## Overview
Formatting, naming, comments, and structural conventions for the codebase.

## Rules

### Indentation and Spacing
- Use 2 spaces for indentation (not tabs)
- Use semicolons at the end of statements
- Maximum line length of 80 characters
- Add trailing commas in multiline objects and arrays
- Use double quotes for strings
- No unused variables or imports allowed

### Naming Conventions
- Use PascalCase for class names
- Use camelCase for variables and functions
- Use UPPER_CASE for constants
- Use descriptive names that convey purpose

### Comments and Documentation
- Add JSDoc comments for all public functions
- Comment complex logic or non-obvious implementations
- Keep comments up-to-date with code changes
- Remove commented-out code before committing

### Structure and Organization
- Keep functions small and focused (single responsibility)
- Group related functionality together
- Separate concerns appropriately
- Use consistent ordering of imports/exports
